# 2026-05-25 session — vllm-moe-marlin CUDA 13 attack

Goal: lever #1 from `../GOAL.md` (Easy wins phase B → vllm-moe-marlin enabling).

Pod: vast contract 37796550, RTX 4090, CUDA 13.0.48, driver 580.82.09,
1766/1524 Mbps inet, Denmark.

## Diagnosis (replicated)

GOAL.md listed three attempted fixes that didn't work:
1. `-Xcompiler -fvisibility=default` on global compile flags
2. `__attribute__((visibility("default")))` on Marlin template decl
3. Same attribute on Marlin template definition

This session ran a minimal CUDA-13 repro on the pod to test
visibility-attribute behavior in isolation (`scripts/cuda13_repro.sh`).
Both "implicit-only" and "explicit-instantiation-separate-TU" cases
linked successfully — confirming the attribute is **respected** in CUDA
13 when on the template definition, but only against a minimal
template. The vendored vllm marlin_template.h has 2000+ lines of
templates and helper functions; one of the helper functions or
non-Marlin templates may be the actual culprit.

## Fix shipped

New file: `crates/ferrum-kernels/kernels/vllm_marlin_moe/kernel_instantiations.cu`

Mirrors the dispatcher's `COMMON_GET_IF`/`BIGGROUP_GET_IF`/... macro
expansion pattern but emits `template __global__ void Marlin<...>` at
namespace scope. This is the same idiom that the (already-working)
`vllm_marlin/` (non-MoE) directory uses, generated by upstream vLLM's
`generate_kernels.py`.

`build.rs::compile_vllm_moe_marlin` now compiles both `ops.cu` and the
new `kernel_instantiations.cu` and bundles them into
`libvllm_moe_marlin.a`. The explicit instantiations live in their own
TU with external linkage, so the dispatcher's `kernel = Marlin<...>`
references resolve at the rust-lld link step.

Instantiation coverage:
- `(half, kU4B8, kFloat16)` — M3 primary (Qwen3-30B-A3B-GPTQ-Int4)
- `(half, kU4, kFloat16)` — AWQ-INT4 fp16
- `(half, kU8B128, kFloat16)` — AWQ-INT8 fp16
- `(nv_bfloat16, kU4B8, kBFloat16)` — bf16 GPTQ-INT4 (future bf16 MoE)
- `(nv_bfloat16, kU4, kBFloat16)`
- `(nv_bfloat16, kU8B128, kBFloat16)`

Each set = COMMON_GET_IF = 40 instantiations × 6 = 240 total
explicit instantiations.
